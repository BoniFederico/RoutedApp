Routed API doc

The following document provides a brief documentation on api endpoints, with the aim of explaining inputs and outputs for each endpoint and some of the error cases that may arise during use.

Endpoints

api/user/register (Method: POST)

Description

It allows a future user to register. The user must provide username, email and password.

JSON Request

{
	"username":"Mario"
	"email":"m@mail.com"
	"password":"pass"
}

JSON Response

200 OK Response

Registration was successful

{
 "Authorization": "Bearer ...",
 "Authorization_Refresh": "Bearer ..."
}

409 Conflict

Another user has the username or the email inserted.

{
 "key": "msg",
 "value": "User already exists!"
}

api/user/authenticate (Method: POST)

Description

Allows a user who already exists to authenticate. Authentication can be done either by email or by username.

JSON Request (username)

{
	"username":"Mario"
	"password":"pass"
}

JSON Request (email)

{
	"username":"Mario"
	"password":"pass"
}

JSON Response

200 OK Response

Authentication was succesfull.

{
 "Authorization": "Bearer ...",
 "Authorization_Refresh": "Bearer ..."
}

401 Unauthorized

Incorrect username, email or password.

{
    "key": "msg",
    "value": "Unauthorized"
}

api/user/refresh (Method: GET)

Description

Allows a user with a refresh token to obtain another access token.

Http Header

It is necessary to provide an Authorization header with a correct refresh token.

JSON Response

200 OK Response

Authentication was succesfull.

{
    "Authorization": "Bearer ..."
}

401 Unauthorized

Expired or incorrect refresh token.

{
    "key": "msg",
    "value": "Token invalid. Please authenticate again!"
}

api/resource/userinfo (Method: GET)

Description

Allows a user with an access token to obtain his information.

Http Header

It is necessary to provide an Authorization header with a correct access token.

JSON Response

200 OK Response

Authentication was succesfull.

{
    "id": "30",
    "email": "prova@gmail.com",
    "username": "Federico"
}

401 Unauthorized

Expired or incorrect access token.

{
    "key": "msg",
    "value": "Token invalid. Please authenticate again!"
}

api/resource/types (Method: GET)

Description

Allows a user to obtain a list of the possible type of a route that he can use.

JSON Response

200 OK Response

Array object with list of route’s types in form of string.

{
    "key": "msg",
    "value": "[{\"id\":1,\"type\":\"Bike\"},{\"id\":2,\"type\":\"Car\"},
							{\"id\":3,\"type\":\"Run\"},{\"id\":4,\"type\":\"Walk\"},{\"id\":5,\"type\":
							\"Train\"},{\"id\":6,\"type\":\"Bus\"},{\"id\":7,\"type\":\"Scooter\"},{\"id\":8,
							\"type\":\"Tram\"},{\"id\":9,\"type\":\"Motorcycle\"},{\"id\":10,\"type\":\"Other\"}]"
}

api/resource/routes/{date1}{date2} (Method: GET)

Description

Allows a user with a correct access token to obtain his routes in the date interval {date1} - {date2}.

Http Header

It is necessary to provide an Authorization header with a correct access token.

JSON Response

200 OK Response

Authentication was succesfull. Response is an array of routes entry.

[
    {
        "id": 50,
        "user": 30,
        "date": "23/07/2021",
        "type": "Bike",
        "route": {...},
        "stages": [
            {
                "name": "San Donà di Piave",
                "lat": 45.6303,
                "len": 12.5661
            }
        ]
    }
]

401 Unauthorized

Expired or incorrect access token.

{
    "key": "msg",
    "value": "Token invalid. Please authenticate again!"
}

api/resource/route (Method: POST)

Description

Allows a user with a correct access token to insert a new route.

Http Header

It is necessary to provide an Authorization header with a correct access token.

JSON Request

The properties route and stages contain serialized object. The property route contain a GEOJSON serialized object.

{
date: "01/01/2022"
route: "{    \"type\": \"FeatureCollection\",    \"features\": 
				[{\"type\": \"Feature\",\"geometry\": {\"type\": \"LineString\",\"coordinates\":
				 [[6.478189,45.380107],[11.294034,43.533823]]},\"properties\": {}}]}"
stages: "[{\"name\":\"Stage1\",\"lat\":1,\"len\":1}]"
type: "Run"
}

JSON Response

200 OK Response

Authentication was succesfull. Route successfully added.

{
  "key": "msg",
  "value": "OK"
}

400 Bad Request

No correct json object has been provided.

{
  "key": "msg",
  "value": "Provided data are not valid!"
}

api/resource/route (Method: PUT)

Description

Allows a user with a correct access token to update an existing route.

Http Header

It is necessary to provide an Authorization header with a correct access token.

JSON Request

The properties route and stages contain serialized object. The property route contain a GEOJSON serialized object.

{
date: "01/01/2022"
id: 50
route: "{    \"type\": \"FeatureCollection\",    \"features\":
				[{\"type\": \"Feature\",\"geometry\": {\"type\": \"LineString\",
		    \"coordinates\": [[6.478189,45.380107],[11.294034,43.533823]]},
				\"properties\": {}}]}"
				stages: "[{\"name\":\"Stage1\",\"lat\":1,\"len\":1}]"
type: "Run"
}

JSON Response

200 OK Response

Authentication was succesfull. Route successfully updated.

{
  "key": "msg",
  "value": "OK"
}

400 Bad Request

No correct json object has been provided.

{
  "key": "msg",
  "value": "Provided data are not valid!"
}

api/resource/route/{id} (Method: DELETE)

Description

Allows a user with a correct access token to delete an existing route with {id} identifier.

Http Header

It is necessary to provide an Authorization header with a correct access token.

JSON Response

200 OK Response

Authentication was succesfull. Route successfully deleted.

{
  "key": "msg",
  "value": "OK"
}

401 Unauthorized

No correct id has been provided.

{
  "key": "msg",
  "value": "Unauthorized access to data!"
}
